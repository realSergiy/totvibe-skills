# Fused-text detector fires on standard markdown links, producing massive false-positive notes.md

## Context

Converting a long blog post (TypeScript 6.0 announcement, ~6000 words) to markdown. The post is link-heavy — dozens of inline markdown links to GitHub PRs, issues, MDN docs, etc.

## Gap

`h2md` reported 77 issues. Every single one was a false-positive "Likely fused text" note triggered by standard markdown link syntax — a URL inside `[text](url)` appearing adjacent to surrounding text. Example from notes.md:

```text
## Likely fused text
**Find:** `mples above now work!\n\n[This change was provided](https://github.com/microsoft/TypeScript/pull/62243) thanks to the work of [Mateusz Burzyńsk`
**Source check:** article.html
**Suggested fix:** verify whitespace between tokens
```

That's perfectly valid markdown: `[This change was provided](https://...) thanks to...`. The 290-line notes.md contained zero actionable items. Reading it was pure token waste.

The actual article.md was clean and correct — no real fused text, no dropped content, all code blocks preserved.

## Responsibility

The fused-text heuristic runs inside h2md and produces notes.md. The agent workflow says "Read notes.md first" — so a high false-positive rate directly wastes agent time. The caller shouldn't need to read 290 lines of false positives to confirm there are no real issues.

## Suggestion

The fused-text detector should exclude content inside markdown link syntax `[...](...)` from triggering. A URL inside parentheses following `]` is not fused text — it's a markdown link. This would have reduced the 77 false positives to 0 in this case.

Additionally, consider suppressing notes.md entirely (or writing an empty/minimal one) when all detected issues are below a confidence threshold. An empty notes.md with a one-line "no high-confidence issues found" is far cheaper than 290 lines of false positives the agent must read and dismiss.

## Impact

Eliminates ~300 lines of wasted context on link-heavy articles — the most common type of technical blog post. Saves one full read cycle and the reasoning to dismiss false positives.
