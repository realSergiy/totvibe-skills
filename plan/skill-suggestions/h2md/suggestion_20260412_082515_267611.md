# Add --copy-to flag and auto-fix stage to reduce agent post-processing loop

## Context

The h2md skill prompt says to use `--save` but the CLI has no such flag. The agent workflow after h2md requires: (1) read notes.md, (2) read article.md in chunks (5+ reads for large articles), (3) write fixes, (4) copy to destination. For this 15k-word article, the post-processing loop consumed ~80k tokens — 80% of the session.

## Gap

The skill prompt suggests `h2md <url> --save bun1_3.md` but h2md ignores the argument. The agent still has to manually copy the file after processing. More importantly, there's no way to ask h2md to apply its own detected fixes automatically.

## Suggestion

Three changes, in order of impact:

### 1. `--copy-to PATH` flag (low effort, high impact)

After all stages complete, copy the final `article.md` to the specified path. This eliminates the manual copy step and lets the agent report the file location immediately.

```text
h2md https://example.com/post --copy-to ./bun1_3.md
```

### 2. `--auto-fix` stage (medium effort, highest impact)

Add a stage between "detect" and final output that automatically applies fixable issues:

- Merge split code blocks (sequential fences with no content between them)
- Apply language detection heuristics to untagged code blocks
- Fix known spacing patterns (Shiki inter-span whitespace)
- Strip clipboard/copy-button artifacts

This is the biggest token saver: the agent currently does all of this manually.

### 3. `--summary` flag for token-efficient agent consumption

Instead of requiring the agent to read the full article.md to verify quality, emit a structured summary:

```yaml
sections: 42
code_blocks: 83
  tagged: 71
  untagged: 12
  languages: {javascript: 45, bash: 18, json: 5, text: 3, toml: 2}
issues_remaining: 3
  - line 76: possible clipboard JS in code block
  - line 234: untagged code block (suggest: javascript)
  - line 891: orphaned label "bunfig.toml" before code fence
```

The agent reads ~200 tokens instead of ~15k tokens of article content, and only dives into sections that need manual attention.

## Impact

Combined, these three changes would reduce post-h2md agent work from ~80k tokens to ~5k tokens — a 16x improvement. The `--auto-fix` stage alone accounts for most of the savings by eliminating the Python-script-writing and manual-editing loop.
