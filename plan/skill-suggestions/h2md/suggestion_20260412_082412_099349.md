# Tabbed UI components (CodeTabs) produce garbled output with interleaved labels

## Context

Same bun.com article. The site uses tabbed code blocks (e.g., install instructions with curl/powershell/npm/brew/docker tabs). The HTML structure is:

```html
<div class="CodeTabs">
  <div class="CodeTabsHeader">
    <div class="CodeTab tabchild-0 active">curl</div>
    <div class="CodeTab tabchild-1">powershell</div>
    ...
  </div>
  <div class="CodeTabItem tabchild-0 active">
    <pre class="shiki"><code>curl -fsSL ...</code></pre>
  </div>
  <div class="CodeTabItem tabchild-1">
    <pre class="shiki"><code>powershell -c ...</code></pre>
  </div>
</div>
```

## Gap

h2md extracted this as flat text: tab labels ("curl", "powershell", "npm") appeared as orphaned words, followed by all code panels concatenated. The agent had to manually reconstruct the tabbed structure as separate labeled code blocks — 6 separate edits just for install sections, plus similar patterns throughout (~10 more tab groups).

Similar patterns appeared with `CodeBlockResults` (command + output in one container) and file-label tabs (bunfig.toml / package.json labels above code).

## Responsibility

Tabbed code blocks are one of the most common patterns in modern documentation sites. Docusaurus (`<Tabs>`), VitePress, Nextra, Bun's site — they all use this pattern. h2md should handle it structurally, not dump it as flat text.

The agent workaround was: read the article, spot the pattern, manually merge/split ~16 code blocks. This is repeatable, deterministic work that belongs in the tool.

## Suggestion

Add a preprocessing step in the extract stage that detects tabbed UI containers:

1. **Detect tab containers** by class patterns: `CodeTabs`, `tabs`, `tab-content`, `tabpanel`, or ARIA roles (`role="tablist"`, `role="tabpanel"`).
2. **For each tab panel**, extract the code block with its tab label as a markdown label:

   ```markdown
   *curl:*
   \`\`\`bash
   curl -fsSL https://bun.sh/install | bash
   \`\`\`

   *powershell:*
   \`\`\`powershell
   powershell -c "irm bun.sh/install.ps1 | iex"
   \`\`\`
   ```

3. **Detect file-name labels** in similar containers (`CodeBlockTab` divs containing filenames like "bunfig.toml", "package.json") and emit them as italic labels before the code block.

4. **Detect command+output containers** (`CodeBlockShell` followed by `CodeBlockResults`) and keep them as a single code block.

## Impact

Eliminates ~15 manual split/merge edits per article with tabbed UI. Saves ~15k tokens of agent editing work and produces structurally correct output on first pass.
