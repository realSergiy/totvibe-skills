# Shiki syntax-highlighted code blocks extract with inter-span whitespace artifacts

## Context

Converting <https://bun.com/blog/bun-v1.3> — a 15,000-word blog post with ~80 code blocks. Bun.com uses Shiki for syntax highlighting. Every code block came out with spaces injected between tokens, making code unreadable and requiring a custom Python cleanup script + 15 manual edits (~50k tokens of agent work).

## Gap

The HTML has this pattern inside `<pre class="shiki"><code>`:

```html
<span style="color: #FF79C6">import</span><span style="color: #F8F8F2"> homepage </span><span style="color: #FF79C6">from</span><span style="color: #F8F8F2"> </span><span style="color: #E9F284">"</span> <span style="color: #F1FA8C">./index.html</span> <span style="color: #E9F284">"</span> <span style="color: #F8F8F2">;</span>
```

h2md extracted this as:

```text
import homepage from " ./index.html " ;
```

Should be:

```text
import homepage from "./index.html";
```

The root cause: Shiki renders string delimiters (`"`, `'`) as separate `<span>` elements with *whitespace text nodes between spans*. When h2md strips HTML tags and preserves text, those whitespace nodes become literal spaces. This affected every single code block (~80 blocks, hundreds of lines).

## Responsibility

This is the #1 source of post-conversion manual work. The agent had to: (1) write a 100-line Python regex script to fix spacing patterns, (2) run it, (3) still do ~15 manual edits for edge cases. That's ~50k tokens of agent work that h2md should handle at extraction time for a few lines of code.

Shiki is used by: Astro, Nuxt, VitePress, Docusaurus, Bun, Deno, Svelte docs, and hundreds of other sites. This isn't a niche issue.

## Suggestion

During the convert stage, when processing `<pre class="shiki"><code>` blocks (or any `<pre><code>` with many child `<span>` elements):

1. **Extract line-by-line from `<span class="line">` elements** (Shiki wraps each line in `<span class="line">`).
2. **For each line, concatenate the `.textContent` of child spans WITHOUT inserting whitespace between them.** The inter-span whitespace in Shiki is purely a rendering artifact — the original code has no such spaces.
3. **Strip `<div class="CopyIcon">` and similar non-content elements** from within code block containers before extraction.

A simpler heuristic that also works: if a `<pre>` element contains >5 inline `<span>` children per line, treat it as syntax-highlighted and concatenate text content without inter-element whitespace.

For language detection on these blocks: check parent container classes. Bun uses `CodeBlockShell` (→ bash), generic `CodeBlock` with no qualifier (→ javascript). Other Shiki sites typically set `data-language` or a class like `language-typescript` on the `<code>` element.

## Impact

Eliminates the single largest source of post-conversion manual work. For this 15k-word article, would have saved ~50k tokens (~50% of the session) and produced usable code blocks with zero manual fixes.
