# Code block language detection defaults to wrong languages

## Context

Bun.com article: ~40 JavaScript/TypeScript code blocks were tagged as `python` in the output. Some others were tagged as `text` when they should have been `bash` or `javascript`. The agent had to write a heuristic detection script and then manually verify the results.

## Gap

The HTML source has `<pre class="shiki">` with no `language-*` class on the inner `<code>` element for most blocks. h2md appears to be guessing the language and getting it wrong — choosing `python` for JavaScript code. This suggests either a broken heuristic or a misread of CSS classes.

The site's HTML does have structural hints that h2md ignores:

- `CodeBlockShell` class → bash
- `CodeBlockPowershell` class → powershell
- Generic `CodeBlock` without shell qualifier on a JS-focused site → javascript
- `language-text` on `<code>` → text/terminal output

## Responsibility

Language detection is a core part of markdown code block generation. Getting it wrong on 50% of blocks is worse than omitting the tag entirely (which would at least not mislabel).

## Suggestion

1. **Check parent container classes**, not just `<code>` classes. Many sites encode the language in wrapper divs (`CodeBlockShell`, `highlight-source-js`, `language-*` on parent).

2. **Use content-based heuristics as fallback:**
   - `import ... from` / `const` / `let` / `=>` / `async` → javascript
   - `$` prompt or `bun`/`npm`/`curl`/`docker` commands → bash
   - `[section]` / `key = "value"` → toml
   - `{"key":` with braces → json
   - `def` / `class X:` / `import X` (no braces) → python

3. **When confidence is low, omit the language tag** rather than guessing wrong. A tagless code fence is better than a mislabeled one — renderers will show it as plain text, and the notes.md can flag it for the agent to decide.

4. **For Shiki blocks specifically**: Shiki's color spans encode semantic info (keywords, strings, operators have distinct colors). The color palette could be reverse-mapped to detect language family, though this is fragile across themes.

## Impact

Eliminates ~40 wrong language tags per article. Saves the agent from writing and running a language-detection script (~5k tokens) and doing manual verification passes.
