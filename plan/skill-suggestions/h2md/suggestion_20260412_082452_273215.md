# notes.md has 68 false-positive "fused text" issues, misses every real problem

## Context

The detect stage reported 68 "likely fused text" issues. After reviewing all 68: zero were actionable. They were long URLs, table cell boundaries, code block content, and anchor links — all valid content. Meanwhile, the actual problems (spacing artifacts in ~80 code blocks, ~16 broken tab UIs, 1 injected clipboard script, wrong language tags on ~40 code blocks) were not detected at all.

## Gap

The agent read all 275 lines of notes.md (burning ~3k tokens), determined every issue was a false positive, then had to discover the real problems by reading the article itself section by section (~25k tokens of reads).

A notes.md that flagged the actual problems — or better, was empty because the tool fixed them — would have saved ~28k tokens.

## Responsibility

The detect stage's value proposition is to save agent time. When it has a 100% false-positive rate and 0% true-positive rate, it's actively harmful — the agent wastes tokens reading it and then ignores it entirely.

## Suggestion

1. **Suppress fused-text detection inside code fences.** Code blocks legitimately contain long strings without spaces (URLs, paths, identifiers). Flagging them is noise.

2. **Suppress fused-text detection for URLs, anchor links, and table content.** These are structured content with known long-token patterns.

3. **Add actual useful detections:**
   - Code blocks with no language tag (or `text` tag) that contain recognizable code patterns → suggest language
   - Code blocks where token spacing looks like Shiki artifacts (`" text " ;` pattern)
   - Orphaned single-word lines between code fences (tab label artifacts)
   - Inline `<script>` content in code blocks

4. **Include fix commands** in notes.md — not just "verify whitespace between tokens" (unhelpful), but actual Edit-tool-compatible old_string/new_string pairs or sed commands the agent can apply directly.

## Impact

Eliminates ~28k tokens of wasted reads + false-positive triage. If detections are actionable with fix commands, the agent can apply them mechanically instead of reading the full article.
