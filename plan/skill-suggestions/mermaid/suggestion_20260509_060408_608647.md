# Description trigger misses noun-phrase mermaid mentions and self-confident skips

## Context
User asked: "please create a plan/board/board.md document with a mermaid diagram showing the project board for all the todo, doing, done tasks". Skill was not invoked. Wrote a flowchart-with-subgraphs board, missed the native `kanban` diagram type that the selection table maps directly to "work-in-progress task board". User flagged this as the exact failure mode the skill exists to prevent.

## Gap
Two trigger weaknesses surfaced:

1. **Noun-phrase "mermaid" doesn't fire.** The description's trigger list is verb-led ("draw/sketch/diagram/visualize/chart/graph"). When "mermaid" appears as a noun-phrase modifier ("a document with a mermaid diagram") rather than a primary verb, the match strength drops. Explicit mention of "mermaid" — in any grammatical role — should be a hard trigger and isn't framed as one.

2. **No guard against confident-skip.** The description says picking the right diagram type matters more than syntax fluency, but doesn't say *what to do when you think you already know the type*. That's the actual failure mode: the model reads "I know flowcharts" and skips the selection table. The skill needs to explicitly name this and tell the model that confidence in the diagram type is itself the signal to verify.

## Responsibility
Skill descriptions are the model's only invocation gate. If the description's trigger conditions don't catch noun-phrase mentions or self-confident skips, no amount of correctness inside the skill helps — the skill never runs. This is on the description, not the caller.

## Suggestion
Two small additions near the top of the description:

- **Hard trigger on the word "mermaid".** Add a sentence like: *"Any mention of 'mermaid' (as verb, noun, or modifier) is a hard trigger — invoke without further judgment."*
- **Anti-confidence guard.** Add: *"If you feel you already know the right diagram type or syntax, that feeling is itself the trigger to invoke — the selection table exists to catch exactly that miscalibration."*

Both are one-liners; total addition ~2 sentences. No restructuring needed.

## Impact
Closes the two failure modes that fired together on this task: weak verb-pattern matching against noun-phrase references, and self-confident skip. Cost is ~2 sentences of description; benefit is the skill actually running on the cases it was designed for.
