# Multi-file SQL queries for cross-file analysis

## Context

Merging two player parquet files (`kaggle_players.parquet` and `sackmann_players.parquet`) by `player_code`. Needed to diagnose why known players (Djokovic, Federer, Cilic) weren't matching — required comparing columns across both files.

## Gap

`peek` SQL mode (`-q`) aliases a single file as `t`. When given multiple files, each is processed independently. I needed cross-file operations:

```bash
# What I wanted (doesn't work)
peek data/prep/kaggle_players.parquet data/prep/sackmann_players.parquet \
  -q "SELECT k.player_name, k.player_code FROM t1 k WHERE k.player_code NOT IN (SELECT player_code FROM t2)"

# What I had to do instead — 3 separate uv run python3 -c calls:
uv run python3 -c "
import polars as pl
k = pl.read_parquet('data/prep/kaggle_players.parquet')
s = pl.read_parquet('data/prep/sackmann_players.parquet')
kaggle_only = k.filter(~pl.col('player_code').is_in(s['player_code'].implode()))
print(f'matched: {k.height - kaggle_only.height}, unmatched: {kaggle_only.height}')
"

uv run python3 -c "
import polars as pl
k = pl.read_parquet('data/prep/kaggle_players.parquet')
s = pl.read_parquet('data/prep/sackmann_players.parquet')
for name in ['cilic', 'djokovic', 'nadal', 'federer']:
    krows = k.filter(pl.col('player_code').str.contains(name))
    srows = s.filter(pl.col('player_code').str.contains(name))
    print(f'{name} kaggle: {krows[\"player_code\"].to_list()}  sackmann: {srows[\"player_code\"].to_list()}')
"
```

## Responsibility

Cross-file comparison (joins, set differences, overlap stats) is a natural extension of parquet inspection — especially during data prep pipelines where you're merging sources. The workaround required importing polars, reading files manually, and writing throwaway scripts. peek already shells out to DuckDB which natively supports multi-table SQL.

## Suggestion

When multiple files are passed with `-q`, alias them as `t1`, `t2`, ... `tN` (in addition to keeping `t` as alias for the first file for backwards compatibility). Example:

```bash
# Set difference
peek a.parquet b.parquet -q "SELECT * FROM t1 WHERE player_code NOT IN (SELECT player_code FROM t2)"

# Join overlap stats
peek a.parquet b.parquet -q "SELECT COUNT(*) as matched FROM t1 JOIN t2 USING(player_code)"

# Side-by-side comparison
peek a.parquet b.parquet -q "SELECT t1.player_code as kaggle, t2.player_code as sackmann FROM t1 FULL JOIN t2 USING(player_code) WHERE t1.player_code IS NULL OR t2.player_code IS NULL LIMIT 20"
```

## Impact

Eliminates the most common reason to fall back from `peek` to raw python one-liners during data prep — cross-file analysis. Saves ~10 lines of boilerplate per comparison and keeps the workflow inside the skill.
